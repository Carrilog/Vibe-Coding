# 任务调度系统

## 概述

NanoClaw 内置定时任务调度器，支持 cron 表达式、固定间隔和一次性任务。任务由 Agent 通过 IPC 创建和管理，在独立或共享会话中执行。

## 任务类型

| 类型 | schedule_type | schedule_value 示例 | 说明 |
|------|--------------|-------------------|------|
| Cron | `cron` | `0 9 * * 1-5` | 标准 cron 表达式，时区感知 |
| 间隔 | `interval` | `3600000` | 毫秒间隔，从创建时开始 |
| 一次性 | `once` | `2026-04-06T09:00:00Z` | ISO 时间戳，执行一次后删除 |

## 任务数据模型

```typescript
interface ScheduledTask {
  id: string              // UUID
  group_folder: string    // 所属分组
  prompt: string          // 执行时的 prompt
  schedule_type: string   // 'cron' | 'interval' | 'once'
  schedule_value: string  // cron 表达式 / 毫秒数 / ISO 时间戳
  context_mode: string    // 'isolated' | 'group'
  script?: string         // 可选的脚本内容
  next_run: string        // 下次执行时间 (ISO)
  last_run?: string       // 上次执行时间 (ISO)
  status: string          // 'active' | 'paused'
  created_at: string      // 创建时间 (ISO)
}
```

## 执行上下文模式

### isolated（默认）

每次执行创建新的 Claude 会话：

```
任务执行 #1 → 新会话 A → 无历史上下文
任务执行 #2 → 新会话 B → 无历史上下文
任务执行 #3 → 新会话 C → 无历史上下文
```

适用于：独立的定期检查、报告生成

### group

复用分组的当前会话：

```
任务执行 #1 → 分组会话 → 有完整对话历史
任务执行 #2 → 分组会话 → 包含 #1 的上下文
任务执行 #3 → 分组会话 → 包含 #1 和 #2 的上下文
```

适用于：需要连续上下文的监控、追踪任务

## 调度循环

```
startSchedulerLoop() — 每 60 秒执行:

1. getDueTasks()
   SELECT * FROM scheduled_tasks
   WHERE next_run <= datetime('now')
   AND status = 'active'

2. 对每个到期任务:
   ├── GroupQueue.enqueueTask(group, task, callback)
   │   └── 等待容器槽位
   │
   ├── runTask(task)
   │   ├── 构建 prompt（含任务上下文）
   │   ├── 选择 session（isolated → null, group → 现有）
   │   ├── runContainerAgent()
   │   ├── 等待 10s（允许最终 MCP 调用）
   │   └── 返回结果
   │
   ├── logTaskRun()
   │   └── INSERT INTO task_run_logs
   │
   └── computeNextRun()
       ├── cron → cron-parser 计算下次时间
       ├── interval → last_run + interval_ms
       └── once → 删除任务
```

## 任务管理 API (通过 IPC)

Agent 通过 MCP 工具管理任务：

```
schedule_task:
  ├── prompt: string        — 任务 prompt
  ├── schedule_type: string — cron/interval/once
  ├── schedule_value: string
  ├── context_mode?: string — isolated/group
  └── script?: string       — 可选脚本

pause_task:
  └── task_id: string

resume_task:
  └── task_id: string

cancel_task:
  └── task_id: string

update_task:
  ├── task_id: string
  ├── prompt?: string
  ├── schedule_type?: string
  ├── schedule_value?: string
  └── context_mode?: string

list_tasks:
  └── (无参数，返回当前分组的任务)
```

## 权限控制

```
Main 分组:
  ├── 创建任何分组的任务
  ├── 管理任何分组的任务
  └── 查看所有任务

非 Main 分组:
  ├── 创建自己分组的任务
  ├── 管理自己分组的任务
  └── 仅查看自己的任务
```

## 任务执行日志

```sql
CREATE TABLE task_run_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id TEXT,
  started_at TEXT,
  finished_at TEXT,
  duration_ms INTEGER,
  status TEXT,        -- 'success' | 'error'
  result TEXT,        -- Agent 输出
  error TEXT          -- 错误信息
);
```

## 时区处理

- Cron 表达式使用配置的本地时区（`TZ` 环境变量）
- 内部存储使用 UTC
- `cron-parser` 库处理时区转换
- 时区验证使用 IANA 标准（如 `Asia/Shanghai`）
