# 容器化与部署

## 容器架构

NanoClaw 使用 Docker 容器隔离每个 Agent 的执行环境。每个对话分组在需要时启动独立容器。

```
┌─────────────────────────────────────────────────┐
│                Docker 容器                        │
│  镜像: nanoclaw-agent:latest                     │
│  基础: node:22-slim                              │
│  用户: node (非 root)                            │
│                                                  │
│  ┌─────────────────────────────────────────────┐│
│  │  全局安装                                    ││
│  │  ├── @anthropic-ai/claude-code              ││
│  │  └── agent-browser (Chromium 自动化)         ││
│  │                                             ││
│  │  Agent Runner (/app/agent-runner/)           ││
│  │  ├── index.ts      — 入口                   ││
│  │  └── ipc-mcp-stdio.ts — MCP 服务器          ││
│  │                                             ││
│  │  Skills (/app/skills/)                      ││
│  │  ├── agent-browser/                         ││
│  │  ├── capabilities/                          ││
│  │  ├── slack-formatting/                      ││
│  │  └── status/                                ││
│  └─────────────────────────────────────────────┘│
│                                                  │
│  ┌─────────────────────────────────────────────┐│
│  │  挂载点                                      ││
│  │  /workspace/group/   ← groups/{folder}/ (rw)││
│  │  /workspace/global/  ← groups/global/   (ro)││
│  │  /workspace/extra/   ← 额外挂载 (按配置)    ││
│  │  /workspace/ipc/     ← IPC 通道       (rw)  ││
│  │  /workspace/.claude/ ← Claude 配置    (rw)  ││
│  │  /workspace/project/ ← 项目根 (仅main, ro)  ││
│  └─────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

## Dockerfile 解析

```dockerfile
FROM node:22-slim

# 系统依赖（Chromium 浏览器自动化所需）
RUN apt-get update && apt-get install -y \
    chromium fonts-liberation ca-certificates \
    git curl jq ...

# 全局工具
RUN npm install -g agent-browser @anthropic-ai/claude-code

# Agent Runner 源码
COPY agent-runner/ /app/agent-runner/
RUN cd /app/agent-runner && npm ci && npx tsc

# Skills
COPY skills/ /app/skills/

# 工作空间目录
RUN mkdir -p /workspace/{group,global,extra,ipc,project,.claude}

# 非 root 运行
USER node

# 协议: stdin 读 JSON，stdout 写 JSON
```

## 容器生命周期

```
1. 启动
   GroupQueue 分配槽位 → buildContainerArgs() → docker run
   
2. 输入
   宿主机通过 stdin 发送 JSON prompt
   
3. 执行
   Agent Runner 调用 Claude Agent SDK
   ├── Claude API 调用（通过 OneCLI 网关）
   ├── 工具执行（文件操作、浏览器、MCP）
   └── 多轮对话
   
4. 输出
   stdout 流式输出（哨兵标记分隔）
   IPC 文件写入（消息、任务操作）
   
5. 空闲等待
   Agent 完成当前任务 → 进入 IDLE_WAITING
   ├── 收到新 IPC 输入 → 继续处理
   └── 30 分钟无活动 → 关闭
   
6. 关闭
   closeStdin() → 容器收尾 → docker stop --time 1
```

## Agent Runner 详解

容器入口 `container/agent-runner/src/index.ts`：

### 输入协议

```json
{
  "prompt": "<messages>...</messages>",
  "sessionId": "session-xxx",
  "groupFolder": "main",
  "isMain": true,
  "taskId": null
}
```

### MessageStream

解决 Claude Agent SDK 的 `isSingleUserTurn` 问题：

```typescript
class MessageStream {
  // 异步可迭代，允许在 Agent 运行期间注入新消息
  // 防止 SDK 在收到第一条消息后就认为对话结束
  
  push(message: string)   // 注入新消息
  close()                 // 关闭流
  [Symbol.asyncIterator]  // 异步迭代器
}
```

### IPC 轮询

Agent 运行期间持续轮询 `/workspace/ipc/input/` 目录：
- 发现新文件 → 读取内容 → 注入 MessageStream
- 实现了容器运行期间的"追加消息"能力

### 输出格式

```
---NANOCLAW_OUTPUT_START---
Agent 的响应文本
---NANOCLAW_OUTPUT_END---
```

### MCP 服务器 (ipc-mcp-stdio.ts)

容器内运行的 MCP 服务器，提供以下工具：

| 工具 | 权限 | 说明 |
|------|------|------|
| `send_message` | 所有分组 | 发送消息到指定聊天 |
| `schedule_task` | 所有分组 | 创建定时任务 |
| `list_tasks` | 所有分组 | 列出任务 |
| `pause_task` | 所有分组 | 暂停任务 |
| `resume_task` | 所有分组 | 恢复任务 |
| `cancel_task` | 所有分组 | 取消任务 |
| `update_task` | 所有分组 | 更新任务 |
| `register_group` | 仅 main | 注册新分组 |

## 构建与部署

### 容器镜像构建

```bash
cd container && ./build.sh
# 等价于: docker build -t nanoclaw-agent:latest .
```

### macOS 服务 (launchd)

```xml
<!-- launchd/com.nanoclaw.plist -->
<plist>
  <dict>
    <key>Label</key>
    <string>com.nanoclaw</string>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>ProgramArguments</key>
    <array>
      <string>npx</string>
      <string>tsx</string>
      <string>src/index.ts</string>
    </array>
    <key>EnvironmentVariables</key>
    <dict>
      <key>ASSISTANT_NAME</key>
      <string>Andy</string>
    </dict>
  </dict>
</plist>
```

安装：
```bash
cp launchd/com.nanoclaw.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist
```

### 日志

```
logs/
├── nanoclaw.log        # 主进程 stdout
├── nanoclaw.error.log  # 主进程 stderr
└── groups/{folder}/logs/
    └── container-{timestamp}.log  # 容器日志
```

## OneCLI Agent Vault 集成

凭证不直接进入容器，而是通过 OneCLI 网关注入：

```
容器内 Agent
    │
    ├── Claude API 请求
    │       │
    │       ▼
    │   OneCLI Gateway (localhost:10254)
    │       │
    │       ├── 注入 API Key
    │       └── 转发到 Claude API
    │
    └── 容器环境变量中无任何密钥
```

## Apple Container 支持

除 Docker 外，NanoClaw 也支持 macOS 原生的 Apple Container：

- 通过 `convert-to-apple-container` skill 切换
- 使用 macOS 原生虚拟化框架
- 更轻量，无需 Docker Desktop
- 网络配置有差异（见 `docs/APPLE-CONTAINER-NETWORKING.md`）
