# NanoClaw 项目概览

## 一句话定义

NanoClaw 是一个轻量级、安全的个人 Claude 助手编排器，通过容器隔离运行 AI Agent，支持多渠道消息接入（WhatsApp、Telegram、Slack、Discord、Gmail）。

## 项目定位

NanoClaw 是 OpenClaw 的极简替代品，核心理念是：

| 原则 | 说明 |
|------|------|
| **小到能看懂** | 整个 src/ 约 8,400 行 TypeScript，单人可完全掌握 |
| **隔离即安全** | Agent 运行在 Linux 容器中，不依赖权限检查，而是 OS 级隔离 |
| **为个人用户设计** | Fork 后通过 Claude Code Skills 定制，不是企业级多租户 |
| **代码即配置** | 不搞配置文件泛滥，直接改代码 |
| **Skills 优于 Features** | 社区贡献以 git branch 形式存在，不是核心功能 |
| **AI 原生开发** | 整个项目由 Claude Code 辅助开发 |

## 核心能力

1. **多渠道消息接入** — WhatsApp、Telegram、Slack、Discord、Gmail，通过 Channel 注册机制扩展
2. **容器化 Agent 执行** — 每个对话组在独立 Docker 容器中运行 Claude Agent SDK
3. **定时任务调度** — 支持 cron 表达式、固定间隔、一次性任务
4. **分组隔离** — 每个群组有独立的文件系统命名空间、会话状态、IPC 通道
5. **安全凭证管理** — 通过 OneCLI Agent Vault 注入凭证，密钥永远不进入容器
6. **流式输出** — 实时将 Agent 响应推送到用户，不是等全部完成再发
7. **远程控制** — 支持从容器内访问宿主机的 Claude Code

## 版本信息

- **当前版本**: v1.2.49（2026-03-26）
- **Node.js 要求**: >= 20
- **许可证**: MIT

## 项目规模

| 指标 | 数值 |
|------|------|
| src/ 源码行数 | ~8,400 行 |
| src/ 文件数 | 34 个 .ts 文件 |
| 测试文件 | 多个 .test.ts |
| 生产依赖 | 3 个（better-sqlite3, cron-parser, @onecli-sh/sdk） |
| 容器依赖 | 4 个（claude-agent-sdk, mcp-sdk, cron-parser, zod） |

## 与 OpenClaw 的关系

NanoClaw 是 OpenClaw 的精简版本，主要区别：

- **规模**: NanoClaw 刻意保持小巧（~8K 行 vs OpenClaw 更大的代码库）
- **安全模型**: 容器隔离 vs 权限检查
- **扩展方式**: Git branch skills vs 插件系统
- **目标用户**: 个人用户 vs 团队/企业
- **定制方式**: Fork + 改代码 vs 配置文件
