# 消息流与数据流

## 完整消息生命周期

从用户发送消息到收到 AI 回复的完整流程：

```
用户在 WhatsApp/Telegram/Slack 发送消息
         │
         ▼
┌─────────────────────────┐
│  Channel 接收消息         │  ← WhatsApp WebSocket / Telegram Bot API / ...
│  onInboundMessage()      │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  storeChatMetadata()     │  ← 更新聊天元数据
│  storeMessage()          │  ← 消息写入 SQLite
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  消息循环 (每 2s)         │
│  getNewMessages(since)   │  ← 获取上次处理后的新消息
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  按分组分发消息            │
│  ├── main 分组: 直接处理  │
│  └── 其他分组:            │
│      └── Trigger 匹配?   │  ← /\bAndy\b/i
│          ├── 是 → 处理    │
│          └── 否 → 跳过    │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  Sender Allowlist 检查    │
│  ├── drop 模式 → 丢弃    │
│  ├── trigger 模式:        │
│  │   ├── 允许 → 触发      │
│  │   └── 拒绝 → 存储不触发│
│  └── allow * → 全部触发   │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  GroupQueue 入队          │
│  enqueueMessageCheck()   │
│  ├── 容器空闲? → IPC 发送 │
│  ├── 容器活跃? → 排队     │
│  └── 无容器? → 启动新容器  │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  formatMessages()        │  ← NewMessage[] → XML
│  构建 Agent 输入          │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  runContainerAgent()                         │
│  ├── buildVolumeMounts()  ← 构建挂载列表     │
│  ├── buildContainerArgs() ← 构建 docker 命令 │
│  ├── docker run ...       ← 启动容器         │
│  ├── stdin.write(prompt)  ← 发送 prompt      │
│  └── 监听 stdout 流       ← 等待输出         │
└────────┬────────────────────────────────────┘
         │
         ▼  (容器内部)
┌─────────────────────────────────────────────┐
│  Agent Runner (container/agent-runner)        │
│  ├── 读取 stdin JSON                         │
│  ├── MessageStream 构建                      │
│  ├── Claude Agent SDK query()                │
│  │   ├── Claude API 调用                     │
│  │   ├── 工具执行 (文件操作/MCP/浏览器)       │
│  │   └── 多轮对话                            │
│  ├── IPC 轮询 (检查新输入)                    │
│  └── 输出: ---NANOCLAW_OUTPUT_START---        │
│           Agent 响应内容                      │
│           ---NANOCLAW_OUTPUT_END---           │
└────────┬────────────────────────────────────┘
         │
         ▼  (回到宿主机)
┌─────────────────────────┐
│  流式输出回调             │
│  ├── 解析哨兵标记         │
│  ├── formatOutbound()    │  ← 移除 <internal> 标签
│  └── channel.sendMessage()│  ← 发送到用户
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  更新状态                 │
│  ├── cursor 前进          │
│  ├── session 更新         │
│  └── 容器进入 IDLE_WAITING│
└─────────────────────────┘
```

## IPC 数据流

```
容器 → 宿主机 (出站 IPC):

容器写入文件到 /workspace/ipc/messages/ 或 /workspace/ipc/tasks/
         │
         ▼
IPC Watcher (每 1s 扫描)
         │
         ├── messages/ → routeOutbound() → Channel → 用户
         │
         └── tasks/ → processTaskIpc()
             ├── schedule_task → createTask() → DB
             ├── pause_task → updateTask(status='paused')
             ├── resume_task → updateTask(status='active')
             ├── cancel_task → deleteTask()
             ├── update_task → updateTask()
             ├── register_group → setRegisteredGroup()
             └── refresh_groups → syncGroups()


宿主机 → 容器 (入站 IPC):

GroupQueue.sendMessage()
         │
         ▼
写入文件到 groups/{folder}/ipc/input/
         │
         ▼
容器内 Agent Runner 轮询读取
         │
         ▼
MessageStream 注入新消息 → Agent 继续处理
```

## 定时任务数据流

```
任务创建:
  容器 IPC → schedule_task → createTask() → DB
  或: 手动创建

任务调度:
  Scheduler Loop (每 60s)
       │
       ▼
  getDueTasks()  ← next_run <= now && status = 'active'
       │
       ▼
  GroupQueue.enqueueTask()
       │
       ▼
  runTask()
       ├── context_mode = 'isolated' → 新会话
       └── context_mode = 'group' → 复用分组会话
       │
       ▼
  runContainerAgent() → 容器执行
       │
       ▼
  logTaskRun() → 记录执行结果
       │
       ▼
  computeNextRun() → 更新 next_run
```

## 分组注册数据流

```
新分组注册:
  Main 容器 → IPC register_group
       │
       ▼
  processTaskIpc()
       │
       ├── 验证权限 (仅 main 可注册)
       ├── isValidGroupFolder() 验证名称
       ├── 创建目录 groups/{folder}/
       ├── 创建 CLAUDE.md
       ├── setRegisteredGroup() → DB
       └── 通知 Channel syncGroups()

分组同步:
  Main 容器 → IPC refresh_groups
       │
       ▼
  Channel.syncGroups()
       │
       ▼
  更新 DB 中的分组元数据
```

## 错误恢复流

```
场景1: Agent 输出后崩溃
  ├── 已发送部分输出到用户
  ├── cursor 回滚到崩溃前
  └── 下次循环重新处理（用户可能收到重复消息）

场景2: Stale Session
  ├── 容器报错 "session not found" 或类似
  ├── 正则匹配检测 stale session
  ├── 清除 session ID
  └── 下次循环创建新 session

场景3: 容器超时
  ├── 硬超时触发
  ├── stopContainer() 优雅停止
  ├── 写入日志到 groups/{folder}/logs/
  └── GroupQueue 标记分组为 IDLE

场景4: IPC 文件处理失败
  ├── 解析或执行失败
  ├── 文件移动到 ipc/errors/
  └── 记录错误日志

场景5: 容器启动失败
  ├── 指数退避重试
  ├── 5s → 10s → 20s → 40s → 80s
  └── 5 次后放弃
```

## Channel 消息格式转换

```
入站 (用户 → Agent):

WhatsApp 消息对象
    │
    ▼
NewMessage {
  sender: string      // 发送者名称
  content: string     // 消息内容
  timestamp: string   // ISO 时间戳
  replyTo?: {         // 引用消息
    id, sender, content
  }
}
    │
    ▼
formatMessages() → XML 字符串
    │
    ▼
Agent stdin


出站 (Agent → 用户):

Agent stdout (带哨兵标记)
    │
    ▼
提取标记间内容
    │
    ▼
stripInternalTags()  // 移除 <internal>...</internal>
    │
    ▼
channel.sendMessage(chatJid, text)
    │
    ▼
WhatsApp/Telegram/Slack 消息
```
