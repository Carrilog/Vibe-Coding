# 技术栈与依赖

## 运行时环境

| 组件 | 版本/说明 |
|------|-----------|
| **Node.js** | >= 20（.nvmrc 指定 22） |
| **TypeScript** | ^5.7.0，目标 ES2022，模块系统 NodeNext |
| **容器运行时** | Docker（默认）或 Apple Container（macOS 可选） |
| **数据库** | SQLite（通过 better-sqlite3） |

## NPM Scripts

```bash
npm run dev      # tsx watch src/index.ts — 热重载开发
npm run build    # tsc — TypeScript 编译
npm run test     # vitest run — 运行测试
npm run lint     # eslint src/ setup/ — 代码检查
npm run format   # prettier --write — 格式化
npm run setup    # tsx setup/index.ts — 初始安装向导
npm run auth     # tsx setup/register.ts whatsapp — WhatsApp 认证
```

## TypeScript 配置

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "sourceMap": true,
    "declaration": true,
    "outDir": "dist",
    "rootDir": "."
  }
}
```

关键点：
- **ES2022** — 支持 top-level await、Array.at() 等现代特性
- **NodeNext** — 使用 Node.js 原生 ESM 模块解析
- **strict: true** — 完整的类型安全

## ESLint 配置

使用 ESLint 9 flat config：
- TypeScript ESLint 推荐规则
- `no-catch-all` 插件 — 禁止 `catch(e)` 不处理错误类型
- 允许 `_` 前缀的未使用变量

## 测试框架

- **Vitest** — 兼容 Jest API，原生 TypeScript 支持
- 测试文件模式：`src/**/*.test.ts` 和 `setup/**/*.test.ts`
- 另有 `vitest.skills.config.ts` 用于 Skills 测试

## 容器镜像

基于 `node:22-slim`，包含：
- Chromium（浏览器自动化）
- 系统依赖（fonts, ca-certificates 等）
- 非 root 用户运行（`node` 用户）
