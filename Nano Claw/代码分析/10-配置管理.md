# 配置管理

## 配置层次

NanoClaw 的配置分为三层：

```
1. 环境变量 (.env)          ← 密钥和运行时配置
2. 外部配置文件              ← 安全策略（存储在项目外部）
3. 代码常量 (config.ts)      ← 默认值和派生配置
```

## 环境变量

### 核心配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ASSISTANT_NAME` | `"Andy"` | 触发词，用于非 main 分组的消息匹配 |
| `ASSISTANT_HAS_OWN_NUMBER` | `false` | 助手是否有独立手机号 |
| `TZ` | 系统时区 | IANA 时区（如 `Asia/Shanghai`） |
| `LOG_LEVEL` | `"info"` | 日志级别：debug/info/warn/error/fatal |

### 容器配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `CONTAINER_IMAGE` | `"nanoclaw-agent:latest"` | 容器镜像名 |
| `CONTAINER_TIMEOUT` | `1800000` (30min) | 容器最大运行时间 (ms) |
| `MAX_CONCURRENT_CONTAINERS` | `5` | 最大并发容器数 |
| `MAX_MESSAGES_PER_PROMPT` | `10` | 每次 prompt 包含的最大消息数 |
| `CONTAINER_RUNTIME_BIN` | `"docker"` | 容器运行时命令 |

### OneCLI 配置

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ONECLI_URL` | `"http://localhost:10254"` | OneCLI 网关地址 |

### 轮询间隔

| 常量 | 值 | 说明 |
|------|-----|------|
| `POLL_INTERVAL` | 2000ms | 消息轮询间隔 |
| `SCHEDULER_POLL_INTERVAL` | 60000ms | 任务调度轮询间隔 |
| IPC Watcher | 1000ms | IPC 目录扫描间隔 |
| Session Cleanup | 24h | 会话清理间隔 |

## 外部配置文件

存储在 `~/.config/nanoclaw/`，项目外部，容器无法访问：

### mount-allowlist.json

控制容器可以挂载哪些宿主机目录：

```json
{
  "allowedRoots": [
    {
      "hostPath": "/Users/me/projects",
      "readWrite": true
    },
    {
      "hostPath": "/Users/me/data",
      "readWrite": false
    }
  ],
  "blockedPatterns": [
    ".ssh", ".gnupg", ".aws", ".azure",
    ".kube", ".docker", "credentials",
    ".env", ".netrc", ".npmrc", ".pypirc",
    "id_rsa", "id_ed25519", "private_key", ".secret"
  ],
  "nonMainReadOnly": true
}
```

### sender-allowlist.json

控制哪些发送者可以触发 Agent：

```json
{
  "default": {
    "allow": "*",
    "mode": "trigger"
  },
  "chats": {
    "specific-group@g.us": {
      "allow": ["trusted-user@s.whatsapp.net"],
      "mode": "trigger"
    }
  },
  "logDenied": true
}
```

## 路径配置

```typescript
// 项目根目录
PROJECT_ROOT = path.resolve(__dirname, '..')

// 数据目录
GROUPS_DIR = path.join(PROJECT_ROOT, 'groups')
DATA_DIR   = path.join(PROJECT_ROOT, 'data')
STORE_DIR  = path.join(PROJECT_ROOT, 'store')

// 安全配置（项目外部）
MOUNT_ALLOWLIST_PATH  = ~/.config/nanoclaw/mount-allowlist.json
SENDER_ALLOWLIST_PATH = ~/.config/nanoclaw/sender-allowlist.json
```

## .env 文件处理

NanoClaw 不使用 `dotenv.config()`，而是选择性读取：

```typescript
// env.ts
function readEnvFile(keys: string[]): Record<string, string | undefined> {
  // 解析 .env 文件
  // 只返回请求的 key
  // 不加载到 process.env
  // 防止密钥泄露到子进程
}
```

## 分组配置

每个分组在注册时可以指定容器配置：

```typescript
interface RegisteredGroup {
  name: string              // 显示名称
  folder: string            // 目录名
  trigger_pattern?: string  // 触发正则
  channel: string           // 绑定渠道
  is_main: boolean          // 是否主分组
  container_config?: {
    additionalMounts?: Array<{
      hostPath: string
      containerPath: string
      readWrite: boolean
    }>
    timeout?: number        // 覆盖默认超时
  }
}
```

## 配置优先级

```
环境变量 > .env 文件 > 代码默认值

示例:
  export CONTAINER_TIMEOUT=3600000  ← 最高优先级
  .env: CONTAINER_TIMEOUT=2400000   ← 次优先级
  config.ts: default 1800000        ← 最低优先级
```
