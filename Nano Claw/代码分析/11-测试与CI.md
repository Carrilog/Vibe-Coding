# 测试与 CI

## 测试框架

NanoClaw 使用 Vitest 作为测试框架，兼容 Jest API，原生支持 TypeScript。

### 配置

```typescript
// vitest.config.ts — 主测试配置
{
  test: {
    include: ['src/**/*.test.ts', 'setup/**/*.test.ts']
  }
}

// vitest.skills.config.ts — Skills 测试配置
{
  test: {
    include: ['skills/**/*.test.ts']
  }
}
```

### 运行测试

```bash
npm run test          # 运行所有测试
npx vitest run        # 等价
npx vitest watch      # 监听模式
npx vitest run src/db.test.ts  # 运行单个文件
```

## 测试覆盖的模块

项目中的 `.test.ts` 文件覆盖核心模块：

| 模块 | 测试文件 | 测试内容 |
|------|----------|----------|
| db.ts | db.test.ts | 数据库 CRUD、迁移、查询 |
| router.ts | router.test.ts | XML 格式化、转义、出站处理 |
| config.ts | config.test.ts | 配置解析、Trigger Pattern |
| group-folder.ts | group-folder.test.ts | 路径验证、安全检查 |
| mount-security.ts | mount-security.test.ts | 挂载验证、blocked patterns |
| sender-allowlist.ts | sender-allowlist.test.ts | 白名单逻辑 |
| timezone.ts | timezone.test.ts | 时区验证、格式化 |
| task-scheduler.ts | task-scheduler.test.ts | 任务调度、next_run 计算 |
| group-queue.ts | group-queue.test.ts | 队列管理、并发控制 |
| env.ts | env.test.ts | .env 解析 |

### 测试辅助

```typescript
// db.ts 提供测试专用初始化
_initTestDatabase()
// 使用内存数据库 (:memory:)，避免文件系统副作用
```

## ESLint 代码检查

```bash
npm run lint          # eslint src/ setup/
```

关键规则：
- TypeScript ESLint 推荐规则
- `no-catch-all` — 禁止不处理错误类型的 catch
- 未使用变量检测（`_` 前缀豁免）

## 代码格式化

```bash
npm run format        # prettier --write
```

配置（`.prettierrc`）：
```json
{
  "semi": false,
  "singleQuote": true
}
```

## CI/CD (GitHub Actions)

### ci.yml — 持续集成

```yaml
# 触发: push 到 main, PR
jobs:
  test:
    - checkout
    - setup node
    - npm ci
    - npm run lint
    - npm run test
    - npm run build
```

### bump-version.yml — 版本管理

自动化版本号递增和 CHANGELOG 更新。

### label-pr.yml — PR 标签

根据修改的文件自动给 PR 打标签。

### update-tokens.yml — Token 计数

使用 `repo-tokens/` Action 计算代码库 token 数量，更新 badge。

## Token 计数

`repo-tokens/` 是一个自定义 GitHub Action：

```yaml
# action.yml
- 使用 tiktoken 计算代码库 token 数
- 生成彩色 badge (SVG)
  - 绿色: <30% 上下文窗口
  - 黄绿: 30-50%
  - 黄色: 50-70%
  - 红色: >70%
- 帮助监控代码库是否"小到能看懂"
```
