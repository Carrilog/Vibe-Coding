# 二次开发指南

## 开发环境搭建

### 前置要求

- Node.js >= 20（推荐 22，见 `.nvmrc`）
- Docker Desktop 或 Apple Container
- Git

### 快速开始

```bash
# 克隆项目
git clone <repo-url> nanoclaw
cd nanoclaw

# 安装依赖
npm install

# 运行安装向导
npm run setup

# 开发模式（热重载）
npm run dev

# 构建容器镜像
cd container && ./build.sh
```

## 项目扩展点

### 1. 添加新的消息渠道

最常见的扩展需求。两种方式：

**方式 A: 使用内置 Skill**

```bash
# 交互式添加
# 在 Claude Code 中运行 /customize
```

**方式 B: 手动实现**

```typescript
// src/channels/my-channel.ts
import { registerChannel } from './registry'
import type { Channel } from '../types'

const myChannelFactory = {
  create(config: any): Channel {
    return {
      name: 'my-channel',
      
      async connect(onMessage) {
        // 连接到消息平台
        // 收到消息时调用 onMessage()
      },
      
      async sendMessage(chatJid, text) {
        // 发送消息到平台
      },
      
      async setTyping(chatJid, typing) {
        // 设置输入状态（可选）
      },
      
      async syncGroups(groups) {
        // 同步分组信息（可选）
      },
      
      async disconnect() {
        // 断开连接
      }
    }
  }
}

registerChannel('my-channel', myChannelFactory)
```

然后在 `channels/index.ts` 添加导入：
```typescript
import './my-channel'
```

### 5. 修改容器挂载

编辑 `src/container-runner.ts` 的 `buildVolumeMounts()`：

```typescript

  hostPath: '/path/on/host',
  containerPath: '/workspace/my-mount',
  readOnly: true
})
```

注意：额外挂载需要在 `~/.config/nanoclaw/mount-allowlist.json` 中配置。

### 6. 修改数据库 Schema

编辑 `src/db.ts`：

```typescript
// 在 initDatabase() 中添加新表
db.exec(`
  CREATE TABLE IF NOT EXISTS my_table (
    id TEXT PRIMARY KEY,
    data TEXT
  )
`)

// 添加迁移（检查列是否存在）
const columns = db.pragma('table_info(my_table)')
if (!columns.find(c => c.name === 'new_column')) {
  db.exec('ALTER TABLE my_table ADD COLUMN new_column TEXT')
}
```

## 关键代码路径

### 消息处理路径

```
src/index.ts:startMessageLoop()
  → src/index.ts:processGroupMessages()
    → src/router.ts:formatMessages()
    → src/group-queue.ts:enqueueMessageCheck()
      → src/container-runner.ts:runContainerAgent()
        → container/agent-runner/src/index.ts (容器内)
      → src/router.ts:formatOutbound()
      → Channel.sendMessage()
```

### 任务调度路径

```
src/task-scheduler.ts:startSchedulerLoop()
  → src/db.ts:getDueTasks()
  → src/group-queue.ts:enqueueTask()
    → src/task-scheduler.ts:runTask()
      → src/container-runner.ts:runContainerAgent()
  → src/db.ts:logTaskRun()
  → src/task-scheduler.ts:computeNextRun()
```

### IPC 处理路径

```
src/ipc.ts:startIpcWatcher()
  → 扫描 messages/ → src/router.ts:routeOutbound()
  → 扫描 tasks/ → src/ipc.ts:processTaskIpc()
    → src/db.ts:createTask/updateTask/deleteTask()
```

## 调试技巧

### 日志

```bash
# 设置详细日志
LOG_LEVEL=debug npm run dev

# 查看容器日志
cat groups/{folder}/logs/container-*.log

# 查看主进程日志
tail -f logs/nanoclaw.log
```

### 容器调试

```bash
# 手动运行容器
docker run -it --rm nanoclaw-agent:latest /bin/bash

# 查看运行中的容器
docker ps | grep nanoclaw

# 进入运行中的容器
docker exec -it nanoclaw-{group} /bin/bash
```

### IPC 调试

```bash
# 查看 IPC 文件
ls -la groups/{folder}/ipc/messages/
ls -la groups/{folder}/ipc/tasks/
ls -la groups/{folder}/ipc/errors/

# 查看快照
cat groups/{folder}/ipc/tasks-snapshot.json | jq .
```

### 数据库调试

```bash
# 直接查询 SQLite
sqlite3 data/nanoclaw.db

# 查看消息
SELECT * FROM messages ORDER BY timestamp DESC LIMIT 10;

# 查看任务
SELECT * FROM scheduled_tasks;

# 查看分组
SELECT * FROM registered_groups;
```

## 架构约束

在二次开发时需要注意的约束：

1. **单进程**: 所有编排逻辑在一个进程中，不要引入微服务
2. **同步 DB**: better-sqlite3 是同步 API，不要在热路径中做大量 DB 操作
3. **容器隔离**: 安全依赖容器边界，不要在应用层做权限检查替代
4. **文件 IPC**: 1s 轮询延迟是设计选择，不要改为 socket（除非有充分理由）
5. **密钥隔离**: 永远不要将 API Key 传入容器环境变量
6. **外部配置**: 安全配置文件必须存储在项目外部
7. **代码规模**: 刻意保持小巧，添加功能优先考虑 Skill 而非核心代码
